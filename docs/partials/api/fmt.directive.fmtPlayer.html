fmtPlayer
directive in module fmt

Description

Includes a configurable player compatible with Flumotion OVRP.

CSS classes

  • fmt-player is set when the directive has been parsed.

Usage

as element:
<fmt-player
       content="{string}"
       [player-html5="{string}"]
       [player-flash="{string}"]
       [tech-order="{string}"]
       [poster-url="{string}"]
       [autoplay-enabled="{boolean}"]
       [on-ready="{expression}"]>
</fmt-player>
as attribute
<ANY fmt-player
     content="{string}"
     [player-html5="{string}"]
     [player-flash="{string}"]
     [tech-order="{string}"]
     [poster-url="{string}"]
     [autoplay-enabled="{boolean}"]
     [on-ready="{expression}"]>
   ...
</ANY>

Parameters

ParamTypeDetails
contentstring

Flumotion ID of the video / audio to reproduce.

playerHtml5
(optional)
string

Select the player to use for HTML5 technology. Options are browser and videojs.

playerFlash
(optional)
string

Select the player to use for Flash technology. Options are strobe, strobe16 and videojs.

techOrder
(optional)
string

Defines the order of technologies using the first supported by the agent. By default it is html5_flash.

posterUrl
(optional)
string

URL of the image to use as a video poster.

autoplayEnabled
(optional)
boolean

Enables the auto-play if true.

onReady
(optional)
expression

Callback function that will be called when the Video.js has been initialised. This function receives the player as a parameter, so you can easily use any Video.js player native method:

               on-ready="onPlayerReady(player)"

Or you can assign the instance directly to the model:

               on-ready="playerInstance = player"

For information about available methods and events please visit the Video.js player API docs
